将博客同步发布到天朝和美帝

##干了三件事

  • 在Mac上安装Hexo
  • 将博客同时部署到Github Page和Gitcafe Page
  • 将域名分别绑定到了两个网站(海内、海外)

##在Mac上安装Hexo
安装很容易，就和在Win上差不多。npm源是不是被半墙了？下载速度很慢，现在这网络环境，唉。

用Hexo进行各项操作时，命令基本都得加sudo，如果不加就会出错。

在用Git推送时，总是提示我SSH有错。反复查询后发现，不仅得把SSH公钥传到Github上，还得把SSH私钥加入到本机SSH客户端。但在Win上使用Git却没有这一步，可能是自动添加了吧，毕竟用的是msysgit集成环境。

##令同一个域名分流海内外流量
我想让wangyufeng.org这个域名，想让国内的流量访问到Gitcafe，海外的流量访问到Github，怎么做呢？

其实很容易，在DNS设置里如图：

主机记录那里我改成了www，这样的话访问wangyufeng.org就可以自动被解析为www.wangyufeng.org，否则必需输入www。

另外，无论是Github还是Gitcafe，都推荐使用CNAME记录。Gitcafe甚至必需使用CNAME，它说

如果您的域名注册商管理界面不支持 CNAME 记录添加，可以使用 DNSPod 或 CloudFlare 等支持 CNAME 记录的第三方域名解析服务来管理您的域名。

为什么呢？Github官网如是说：

We strongly recommend that you use a custom subdomain for these reasons:

  • It gives your GitHub Pages site the benefit of our Content Delivery Network.
  • It will not be affected by changes in the underlying IP addresses of GitHub’s servers.
  • Pages will load significantly faster because Denial of Service protection can be implemented more efficiently.

大意是

  • 有CDN给你加速
  • 就算我们挪窝了你页面也能用
  • 更好的DOS保护

所以，如果不是弄这种流量的分流，正常情况下一条CNAME解析就足够了。
现在网上的绝大多数教程/帖子都是A记录，还有A记录和CNAME记录混用的。

这件事儿带给我的收获是，多看看官网的东西肯定没错。看网上的经验贴前也得注意发帖日子。

毕竟不是所有的东西都像C语言那样，(近似于)静静地待在角落里一动不动，一声不吭。

如果大家对Hexo建博客感兴趣，不妨搜索一下Github Hexo 博客这样的关键字，教程非常多。很好玩。
