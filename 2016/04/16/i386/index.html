32位还是64位？ | 七系

发表于 2016-04-16 | 更新于 2016-09-03

这是 MySQL 下载页面列表的部分截图，可以发现名字同样的软件，分为了 amd64 和 i386。

同样的，还有一系列词汇，例如

  • x86
  • x86_64
  • i386
  • i586
  • i686
  • amd64
  • x64

他们到底是什么意思，有什么区别呢？

此文分析一下。如果有错误还请指出

##1. 关键在于是32位还是64位

这些名词说明了软件环境的架构版本。

这些名词可以分为两组：32位和64位。

CPU 和操作系统有32位和64位的区别。比如，我们常说，这个电脑是64位的，可能是说这个 CPU 是64位架构的。从计原课可以得知，32位的 CPU 意味着这个 CPU 有32位宽的数据线、地址线、控制线。常说，32位的电脑最大只能用不到4GB的内存，就是因为地址线最大只能是32位。

操作系统也有32位和64位的。64位的CPU可以运行64位的 Windows 7，一般情况下也能运行32位的 Windows 7，但是反过来就不行。即32位CPU不能运行64位的操作系统。后文会简述原因。

也就是说，在下载、安装软件包之前，我们需要先判断自己的操作系统的版本是32位还是64位（不需要判断CPU的位宽，因为软件安装到操作系统上，是以操作系统的版本为准）。

这些名词中中，代表32位的有：

  • x86
  • i386
  • i586
  • i686

64位的有

  • x86_64
  • amd64
  • x64

举例：如果我知道我的操作系统是64位的Win7，想安装一个64位的软件，只要软件安装包名字中有’x86_64’, ‘amd64’, ‘x64’中的任意一个就可以了。

通常情况下，64位的操作系统也可以安装32位的软件，但是反过来不行。说明64位架构的向下兼容性很好。后文还会提到向下兼容型。

##2.为何这么叫？

问得好。

###32位系列

####x86

INTEL(英特尔)公司最早生产的那些CPU，都是以数字命名的，什么8086啊，80286，80386什么的。所以就叫 x86 了。

后来INTEL惊喜地发现美国的法律不允许以纯数字作为商标，所以才没这么继续下去。

####i386, i586, i686

这里「i」就是指 INTEL，i386是 INTEL 早年的一种CPU的架构，名字源于80386。几乎适用于所有的x86平台。也就是说i386的通用性很强。

i586, i686是升级版。大家下软件的话，可以大胆选择i686，因为奔腾2以后的32位CPU就都是i686了。奔腾2是什么概念呢？只要这电脑是二十一世纪生产的，就肯定可以用i686了。

那如果只有i386怎么办？没关系，i686是向下兼容的，也就是说，i686架构的CPU，是可以支持i386的软件的。同理，既然64位CPU也向下支持32位CPU，所以64位CPU也能运行为i386架构设计的软件。

###64位系列

####x86_64, x64

x86_64也很常见，这意味着64位架构。但是x86_64对32位软件的支持也很好，也具有向下兼容性。从名字能看出来，x86_64是对x86架构的64位拓展。

x64则是x86_64的简称。

####amd64

也是64位架构的意思。为什么有个amd呢？大家知道AMD和INTEL是美国两家最大的CPU制造商，当然了，AMD一直是小弟。INTEL公司财大气粗，都不用换股，直接用现金储备就能把AMD给买了。不买的原因主要是考虑美国和世界各国的反垄断法。

虽然说AMD一直生活在INTEL的阴影下，不过人家也牛逼过呀，有人说AMD是「最伟大的失败者」，因为它一直在跟一个垄断者陪玩。有一阵子技术特别厉害，比如你看，是AMD率先推出64位架构的CPU。当然咯，后来INTEL也进入64位市场了，这个名字就逐渐被人们用’x86_64’替代。而’amd64’的使用，属于习惯问题。比如就是有人习惯把首尔叫做汉城，对吧。

到了今天，INTEL的垄断没有那么强大了，因为移动设备崛起了。移动设备用ARM架构更加普遍，比如高通公司就是造ARM芯片的大厂，现在也很牛的。

###IA-32和IA-64是什么意思？

其实上面那些词，完全可以笼统分为两个，即x86和x86_64，代表32位和64位。但他们都属于IA-32架构，即INTEL 32位体系架构。正因为如此，x86_64对x86的兼容性很好，体现在32位的程序基本都能运行在64位的电脑上。

但是还有一种真·64位架构，即IA-64, 用IA-64架构的CPU（例如INTEL的「安腾」，你听说过吗？反正我没听说过），最大的问题就是兼容性不好。虽然技术更先进，但是兼容性不好的东西，往往是不会被市场接受的。正因为对兼容性的需求，才有的x86_64架构。

计科的计原课学的MIPS，也是分MIPS32和MIPS64架构。咱们学的是MIPS32. 如果我没记错的话，这两种架构也是相互不兼容的。

君不见，当年IBM-PC一统天下，靠的不还是兼容性。IBM-PC属于兼容机，而那些不生产兼容机的PC厂商，到今天基本连渣都不剩了。

微软的 Windows 也是兼容性特别强的软件。Win98年代的红色警戒2，到了Win10上也能(勉强)运行。我觉得这简直是个奇迹。

看，兼容性是多么重要啊。

##总结

总之，32位的操作系统就装32位软件，64位的操作系统优先装64位的软件，没有64位的话装32位的也可以。
