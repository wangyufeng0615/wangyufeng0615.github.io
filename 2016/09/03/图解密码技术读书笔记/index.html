《图解密码技术》读书笔记

发表于 2016-09-03 | 更新于 2016-10-10

通过这本书，捋了一遍密码技术中的各种名词，有些在以前不理解其含义的词，现在大体有个概念了，开心。

‘图灵程序设计丛书’每一本都很喜欢。在读这本书的时候，隐约想起了《信息简史》这本书中的一些内容，比如信息的压缩、信息的意义等等。等时机成熟，再重读一遍《信息简史》。

另外，”密钥”这个词真的是读”密yue”吗，感觉好奇怪啊=。=

密码

密码与信息安全常识

  • 不要使用保密的密码算法，即，不要依赖于”隐蔽式安全性”。公开的算法是最安全的，算法的效果取决于”种子”。
  • 与其使用低强度的密码，不如不使用密码。
  • 没有永恒安全的密码。
  • 密码只是信息安全的一部分(e.g. 社工)。

传统密码

  • 凯撒密码：字母表平移。可暴力破解。
  • 简单替换密码：字母表映射。可通过频率分析破解。
  • Enigma。Enigma的设计就不依赖于”隐蔽式安全性”，即，机器被敌军捕获了也没关系。有一段阿兰·图灵的故事。

对称密码

  • 对称密码：加密密钥和解密密钥一样。
  • 一次性密码本：从数学理论上就绝对无法被破解的密码，量子计算机也不行。但没被实际应用，原因是密钥配送和密钥长度是问题。不能被破解的原因：解密文本包含原文的所有排列组合空间。
  • DES：分组密码的一种。结构是 Feistel 网络。已经弃用。
  • 三重 DES：DES 的加强版。
  • AES：AES所采用的算法：Rijndael。AES 是对称密码的首选。

分组密码

  • 分组密码：每次只能处理特定长度的一块数据的一类密码算法。
  • 模式：
    • ECB（不要使用）
    • CBC
    • CFB
    • OFB
    • CTR

公钥密码

  • 公钥密码的诞生：解决”密码配送问题”，即，密钥的传输过程中如何保证不被窃听。
  • 密码配送问题的解决方案：
    1. 通过事先共享密钥
    2. 通过密钥分配中心
    3. 通过 Diffie-Hellman 密钥交换
    4. 通过公钥密码
  • 公钥密码的含义：Alice 想要发送秘密的信息给 Bob，但是密钥的发送不能保证安全性(密钥也需要加密，但加密密钥的密钥保证不了安全)，于是 Bob(接收者) 就生成一个密钥对（公私钥），把公钥发给 Alice，这个公钥全世界都能看。Alice 用 Bob 的公钥把信息加密，发给 Bob。Bob 再用自己的私钥解密这个消息。因为 Bob 私钥根本就没进行”发送”，一直在自己手上，所以即便密文被被窃听，窃听者也解密不了，因为没有私钥。
  • 公私钥之间具有紧密的数学联系。
  • 一旦发现了对大整数进行质因数分解的高效算法，RSA 就被破解。
  • P=NP？ <—> 具有该高效算法
  • RSA 的不足：中间人攻击。通过证书解决。

混合密码系统

  • 由于公钥密码的速度比对称密码慢很多，所以解决方案是：用对称密码加密明文，然后将对称密码的密钥用公钥密码加密。

认证

单向散列函数

  • 求比特序列的 Hash 值。即，序列的特征值，只能单向运算（不能逆推），计算速度快，一般长度固定。也叫作指纹。
  • 不同的消息散列值也不同，除非出现碰撞。
  • MD5（安全性已不能保证）
  • SHA-2
  • SHA-3(最新，使用了和 SHA-1和 SHA-2 不同的结构，即 Keccak)
  • 对单向散列函数的攻击：
    • 暴力破解（碰撞，对抗弱抗碰撞性）
    • 生日攻击（碰撞，对抗强抗碰撞性）

消息验证码

  • 简称 MAC，Message Authentication Code
  • 是一种与密钥相关联的单向散列函数
  • 双方需要共享密钥
  • 可以使用单向散列函数实现，也可以使用分组密钥实现
  • 对消息验证码的攻击：重放攻击、密钥推测攻击。
  • 消息验证码的缺点：不能对第三方验证（证书可解决），不能防止”否认”。

数字签名

  • 即，将 RSA 的过程反过来。用私钥加密自己的信息，所有人可用公开的公钥来验证。
  • 数字签名的方法：对消息进行签名(量大，且机密性不是数字签名的需求)。所以使用：对散列值进行签名。
  • “数字签名”一词中的”签名”，是”signature”，不是”autograph”.
  • 对数字签名的攻击：中间人攻击、对单向散列函数攻击、利用数字签名攻击公钥密码（所以绝对不要对不明来源的信息进行签名）、潜在伪造。
  • 为了确认自己用于验证数字签名的公钥是否合法，需要使用证书。

证书

  • 即，证书颁发机构用自己的私钥，对一个需要验证的公钥进行数字签名，并发布自己的公钥。
  • 信任链递归了怎么办？Root CA。理论上，如果能够取得可信的公钥，就不需要认证机构。即，
  • 公钥基础设施（PKI）
  • 如果证书作废，发布 CRL。作为用户一定及时更新 CRL。

第三部分

##密钥

  • 密钥的本质：一个巨大的整数
  • 分类
    • 对称密码的密钥、公钥密码的密钥
    • 消息认证码的密钥、数字签名的密钥
    • 用于确保机密性的密钥、用于认证的密钥
    • 会话密钥、主密钥
  • 内容加密密钥：CEK
  • 密钥加密密钥：KEK
  • 生成密钥：使用专门针对密码学设计的伪随机数生成器
  • 基于口令的密钥：最后一层保障放在大脑里
  • 密钥的配送（e.g. RSA）
  • 密钥的更新（e.g. 散列值递归，具备”向后安全”）
  • 密钥的作废：CRL

如何生成安全的密钥：

  • 不使用自己的信息、不使用别人的有关信息、不使用逻辑上顺序的一系列密码。

随机数

随机数发生器对密钥的生成很重要。

随机数的特性：

  • 随机性
  • 不可预测性
  • 不可重现性

只具有随机性的是弱伪随机数、具有随机性和不可预测性的是强伪随机数、具有三者的是真随机数。

  • 软件生成的随机数只能是伪随机数。真随机数需要靠物理传感器等硬件设备生成。

线性同余法：具有随机性，但是不具有不可预测性，所以不能用于密码学的随机数。c 语言标准库中的 rand() 就使用的是线性同余法。

生成伪随机数列的方法：

  • 散列法
  • 密码法

伪随机数生成器的种子：真随机数。

PGP

  • Pretty Good Privary, 一个包含各类密码工具的密码软件。
  • 采用”信任网”，不依赖特定的认证机构。去中心化。

SSL/TLS

  • 结合对称密码、公钥密码、单向散列函数、消息认证码、伪随机数发生器、数字签名等技术。
  • 对 HTTP 协议的保护，即 HTTPS
  • 也可以用于保护其他协议
  • 心脏出血漏洞：利用的是 OpenSSL 对 TLS 实现上的缺陷。

密码技术与现实社会

###小结

  • 对称密钥是一种用相同的密钥进行加密和解密的技术，用于确保消息的机密性，目前对称密码的算法主要使用 AES。但存在密码配送问题，用公钥密码解决。
  • 公钥密码是一种使用不同的密钥进行加密和解密的技术，同样用于确保消息的机密性。速度很慢，一般和对称密钥组成混和密码系统。但存在中间人攻击的问题，用数字签名和证书解决。
  • 单向散列函数是一种将长消息转换为短散列值的技术，用于确保消息的完整性。主要使用 SHA-2和 SHA-3.
  • 消息认证码是一种能够识别通信对象发送的消息是否被篡改的认证技术，用于验证消息的完整性，以及对消息进行验证。常用的算法是 HMAC。
  • 数字签名是一种能够对第三方进行消息认证，并能够防止通信对象作出否认的认证技术。算法包括 RSA 等。公钥基础设施（PKI）中所使用的证书，就是对公钥加上认证机构的数字签名所构成的。要验证公钥的数字签名，需要通过某种途径获取认证机构自身的合法公钥。

  • 伪随机数发生器是一种能够生成具备不可预测性的比特序列的技术，由密码和单向散列函数等技术构成。

  • 密码技术的框架化：可方便地替换掉不再安全的密码技术。

  • 密码技术实际上可以看成压缩技术。

关于比特币：

  • 区块链
  • P2P 网络
  • 挖矿
  • 椭圆曲线

关于完美：

  • 没有完美的密码技术。即使有完美的密码技术，使用密码技术的也是不完美的人。完美的安全性是无法达到的。

椭圆曲线：

  • 既不是”椭圆”，也不是”曲线”(是离散值)
  • 椭圆曲线上的运算相当于在包含椭圆曲线上所有点以及无限远点的集合上的阿贝尔群，群以无限远点 O 为单位元。（已经忘记阿贝尔群是啥意思了 Σ(°Д°; 罪过啊）
