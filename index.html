在工大上网：共享网关账号还是各用各的？

发表于 2017-03-27

摘要：宿舍上网常见两种模式，“用一个人的账号”和“各自用各自的账号”，本文讲解现象背后的原因。

前言：如果遇到不懂的名词，没有关系，不影响你理解本文想要讲明的主要内容。酌情跳过扩展阅读，善用百度。

<!—- more —>

你所在的宿舍是怎样上网的呢？有些宿舍是所有人用一个账号，典型情景是这样的：每天第一个起床（或第一个回宿舍）的人登陆自己的网关，然后其他连接到 Wi-Fi 上的人也能上网了。这样的话，网关账号流量容易消耗的很快，没两天就用没了。

另外一种情景，就是每个人都登陆自己的账号，每台电脑、每个手机都需要单独登陆网关账号。

以上提到的两种情景，对于联通光纤来说也是一样的：有些宿舍共用一个联通账号，有些是各自买各自的。

那么，为什么会有这种情况呢？对于绝大多数宿舍来说，宿舍都买了路由器。可以观察一下路由器的接线，当从墙上网口接出来的网线，插到路由器的 WAN 口时，就是共用一个网关账号；当插到任一一个 LAN 口时，就是每个设备都分别登陆一个网关账号。大家可以根据宿舍成员的需要改变相应的设置。

图片来源：Wikipedia

WAN 口和 LAN 口长什么样呢？参照图片，看路由器的后边，单独一个的那个口就是WAN口，并排的几个网口（根据路由器有所不同，我见过有2-4个的）就是 LAN 口。这个图上没标”WAN”和”LAN”，一般路由器都会标注的很清楚。
为什么把一端插在墙上网口的网线，另一端插在 WAN 口和 LAN 口的作用会不同呢？咱们先从家里上网说起。

对于绝大绝大多数的家庭，路由器的使用方法很简单：把「猫」的网线连到路由器的「WAN」口上，然后把台式机、笔记本等等连到「LAN」口上，手机等无线设备通过Wi-Fi连到路由器。

也就是说，WAN 口相当于是路由器连接外网的口，可以理解为路由器的「大门」，而 LAN 口是连接「内部的设备」用的。连接到路由器 LAN 口的设备，包括那些通过 Wi-Fi 连接到路由器的无线设备，在这个路由器的内部，构成了一个小局域网。而这些设备们共同的出口，就是路由器的WAN口。对于WAN口来说，有这样一个特性：对于路由器的外界来说，路由器的「大门」是不透明的，外界看不到这个路由器背后有什么设备，只把路由器当做一个整体看待。

现在规范一下用词，相比「大门」，更严谨的说法是「网关」。那么，怎么理解刚才提到的「不透明」呢？可以把网关理解为传达室老大爷，你要是想找大院内的什么人，你不必直接找到那个人，把你要传的话和要接收这个消息的人告诉老大爷就行，老大爷会代你传达。这个大院对于大院外部的人，就是「不透明」的，他们能看到的只是传达室的大爷。这个大爷就是「网关」。而路由器的 WAN 口就是路由器的网关。

咱们回过头来，先分析第一种情况，即为什么墙上引出的网线插到 WAN 口，大家就共用一个网关账号？因为这时候，对于学校的校园网来说，路由器作为一个整体存在，校园网只能看到宿舍里这台路由器的「网关」。实际情景是：我们通过一个设备，比如手机，连到这个路由器并登陆网关时，我们是替这台路由器的「传达室老大爷」注册了网关，而连接这台路由器的所有设备，都通过老大爷跟外界交流，并不和外网，也就是学校的网关有直接的接触，所以大家就共用同一个账号了。

说到这，你也许有些迷糊，会问：为什么出现了好多个网关？是这样的，咱们学校的校园网是一个大局域网，有一个总的网关，也就是我们登陆网关账号的那个网关。而我们的路由器，构成了一个「小局域网」，也有属于自己的网关（也就是 WAN 口）。我们做的事情是：把这个小网关通过一个网关账号，登陆到了学校的大网关上。

也就是说，上文中所有提到的「网关账号」，网关指的都是校园网的大网关。

拓展阅读：能注册学校网关的设备，是以 IP 地址为单位的。如果是用刚才提到的插 WAN 口的方式上网，相当于路由器的网关有一个属于自己的IP地址，对于外界来说，这个路由器只以这个 IP 存在。而路由器背后的每个设备，确实都有自己的 IP 地址，但这些地址都是路由器这个小局域网内的内网 IP 地址，对于外界来说是不可见的。

既然如此，咱们继续说第二种情况，即如果把网线插到路由器的任意一个 LAN 口上，为什么就可以实现每个设备都单独注册一个网关账号。

如果把网线插到 LAN口 上，其实路由器的功能就「退化」了，退化成了一种叫做交换机的设备。也就是说，现在路由没有了「网关」（因为根本没用到 WAN 口），现在传达室老大爷不存在了，而连接到路由器的每台设备，不管是通过网线（电脑插到其他 LAN 口）还是 Wi-Fi 连接到路由器，都是作为单独的个体暴露在外网中。

换言之，现在这台路由器的作用，就是把墙上的一个网口，扩展为若干网口，同时加入了无线功能。但实际上，就相当于连接到这台路由器的每个设备，都单独地连接到墙上的某一个网口。但是由于宿舍墙上的网口数量很有限，所以我们使用路由器的这种功能，拓展网口数量并加入无线功能，而每个设备都是独立接入校园网的。

对于这种情况，如果想象不出连线的画面，可以参考上面那个图，即，把从墙上引出的网线插在 LAN 口1上，然后把电脑连到 LAN 2到4口上，就可以实现每台设备单独注册网关账号的特性。

至此，我们知道了为什么有些宿舍上网的方式是所有人共用一个账号，有些宿舍是各自登陆各自的账号。

拓展阅读：最后，我分享一下我所在宿舍的网上方式：我们共同买了联通光纤的包学期卡，一人一学期不到50块钱，所有人共用一个账号。实际下载速度最大6M/s（相当于50Mbps）。由于宿舍楼 Wi-Fi 信号太多，感觉无线信号质量很不好，所以我们宿舍的电脑都使用网线连接路由器。但是一个路由器只能有线连4个电脑（因为只有4个LAN口），6个电脑怎么办呢？我们将两个路由器进行有线桥接（方法是：路由器2的LAN口接路由器1的LAN口，关闭路由器2的DHCP，将LAN IP设置为同一网段），这样的话，相当于扩展了路由器。网络质量又好又便宜。

最后上一张无线信号干扰的示意图，即 2.4G 频段的三个独立信道，每个信道上都会有很多信号，在这种情况下，即便 Wi-Fi 信号满格，信号质量也会下降很多。宿舍楼的实际情况要更严重。

我的2016

发表于 2016-12-31

2016年过去了，我很怀念它。

  1. 一年下来，偶尔打打篮球，又学了足球，挺好。我很感谢这两样运动，让我对自己有了许多思考（以及吸入了许多雾霾）。

因为一次篮球赛弃赛，让我想了很多。

我记得《纸牌屋》里有这样一句台词（大意）：“考验一个人的品性，不是在于他享受胜利果实时的表现，而是他承受失败的苦涩时的表现。”我对此的理解是，在逆境和困难之中，体现一个人的品性。

篮球比赛，乃至任何竞技运动，对手有强有弱。有些对手已经强大到，己方几乎没有任何胜利的可能性。但如果因为对手的强大而不去拼搏，那竞技运动的意义又何在呢？技不如人是常事，但精神的拼搏，没有认输的理由。

如果我是一个军人，我愿意像《亮剑》中的骑兵连连长那样牺牲，我想那是一个战士的荣耀。

明年可能是我最后一个能待在校园里的年份了，一定要好好锻炼身体:)

  1. 大二的体育选修，我报的是足球，报名之前我犹豫了。因为我并没有足球基础，如果报了足球，分数想必会比较低（的确不高，第一学期70多分）。如果我继续报大一已经上了一年的篮球，虽然我篮球打得不咋地，但凭着基本功，也能轻松95+。

这是一个跳出舒适区和待在舒适区中的选择。那么多鸡汤不能白喝，于是我选择了跳出舒适区，报了足球。正如我所预料的，学起来很不容易，刚开始的那几节课，下课以后脚都很疼（不会发力），没有带球的基本功，课上比赛时也很难触球。

但经过了一年的体育课，虽然足球踢得还是很渣，但基本的传接球还可以，也终于会一点开大脚了。还替舍友参加了一场足球赛，凭着我耐力好能跑的特点，防对面箭头的效果还不错。

虽然我损失了不少加权分数，但我觉得这个跳出舒适圈的选择太值了：让自己入门了一项新的体育运动、体会到了在足球赛中上场的感觉。

另外，如果你承认的话，我觉得打守望先锋也算是竞技的一种。玩这个游戏玩的不多，但收获了快乐，以及磨炼了自己容易着急的心性。个人经验：放低姿态，团队配合，享受的心态，才能感受到游戏带给人的快乐，否则容易消耗精力。

3.

暑假的时候去实习了，详细的感想我已经写在博客里了。总之，我感觉知道创宇是一家非常好的公司。图是望京SOHO，创宇的办公地点。

这次实习让我从象牙塔中走出了一小步，感受了工作的环境，促使我思考了很多关于未来发展的事情，比如要不要读研等等。读研这个事，我到今天也没有给自己一个明确的答案，不过还有半年的时间让我来探索。

总之，我觉得自己挺享受工作的氛围。即便我最终读研究生了，我想我不会是因为逃避就业而做出的选择。

大三伊始，我收获了一枚小可爱，我的女朋友牛畅。能遇到她，我想我很幸运。她是一个很优秀的人，比很多人想象的更加优秀；她也是一个很普通的小姑娘，比很多人想象的更加普通。

在这一年中，我探索到了一个自己的兴趣点，就是把某种事物用自己的嘴讲出来。技能师承我爸，哈哈。大概一共讲了三四次？有C语言也有Office。

6.

我回顾我的2016，我觉得自己在某些方面变懒了，在某些方面变勤奋了。

学习方面划水的现象非常严重…归结起来，就是不求甚解+犯懒。这样不好。而且看书的进度也变慢了，其实是因为注意力集中的能力下降了。

总的来说，我觉得自己的2016年，对时间的利用效率很不好，而且经常三分钟热度。许下新年愿望：希望2017年能变得更加持之以恒一些。

寒假的时候我打算通过大量阅读，恢复一下专注的能力。书单真的已经积攒到非常可观的程度了。

至于变勤奋的地方，我觉得我有意识地让自己“跳出舒适圈”，去探索自己的可能性边界，体验不一样的东西。

比如有一个周末，我参加了雷励IW活动，真真是“跳出舒适圈”，让自己有了一段独特的经历。很不错的活动。在这个活动中，我最大的收获就是对自己性格，尤其是在团队中的性格，有了更多的思考以及和他人交流的机会。

《智能时代》读书笔记

发表于 2016-10-18

欢迎大家关注「七系」、转发文章。

阅读本文大约需要 7 分钟。

刚刚过去的美国总统选举我关注了一下，发现硅谷的IT产业基本都是支持希拉里的，为什么呢？根据网上的分析，其中的一个原因是，特朗普强调要增加工作岗位、让工人们找到工作。但如今的趋势是机器越来越多地替代人的工作，比如AlphaGo就是一个里程碑一样的事件：机器不仅可以替代人类做机械的重复工作，也可以替代人类完成“智力活动”。而硅谷的一些公司，如Facebook、Google等，自然是人工智能产业的先锋，他们做的事情是让AI代替人力，而这正和特朗普的政治主张向左。假设：一个卡车司机，他是更支持自动驾驶技术，还是更支持一个能保证自己工作的人呢？我想无疑是后者。

我于两个月前看了吴军的《智能时代》一书，其中就描绘了这种随着人工智能的发展，技术与社会之间可能产生的冲突。吴军老师的书我每本都追着看，这本《智能时代》中的一些观点在其作品《浪潮之巅》以及其他描绘大数据产业的书中也有看到过，但我想这仍是一本好书。而且这本书给我的一个感觉是，作者不仅仅是在描述表面的现象，而是从很基础的内容逐渐推演出观点，并逐渐和现实相结合。感觉看书的过程，也是一个自己思考的过程，不仅仅是获取信息。

记录一些零碎的笔记和摘要吧。相信过些时日我再回顾自己的笔记，或再看一遍本书时，也会有新的收获和感悟。我看《信息简史》这本书就是，高中的时候读，大一的时候读，大三的时候读，完全是不同的理解和感悟。

《图解密码技术》读书笔记

发表于 2016-09-03

工大野生助手

发表于 2016-07-24

工大野生助手

https://github.com/wangyufeng0615/bjuthelper

访问量

预览


功能

  • 查某学期的各科成绩。
  • 查所有学期总的加权平均分和平均GPA。

运行模式

  • 这两个PHP页面运行在Apache中；
  • Apache安装在我的电脑上；
  • 电脑连了学校的内网，也就是bjut-wifi或者插网线；
  • 将自己电脑的80端口映射到外网。映射的方法很多，我用的是ngrok这个开源工具，也可以用花生壳等。
  • 【手机端(扫二维码访问登陆页面)】<—>【外网映射服务器】<–(学校网关)–>【内网电脑,运行着Apache】<—>【教务网站】

细节

  • login_grade.php负责登陆，require_grade.php负责具体的查询。

login_grade.php

  • UI用的是「WeUI」，微信团队做的，和微信的视觉效果一致；
  • 每个学校的正方教务都不太一样，我的代码中的各种http header参数、post参数基本符合北工大的教务网站。
  • 教务系统的默认登陆页面是default2，可以换换default后面的数字，可以有不需要验证码的登陆页面。

require_grade.php

  • 中文编码容易出现乱码，涉及到本地IDE的编码、PHP的编码和网站的编码；
  • 有两种post，第一种是登陆，第二种是获取成绩信息。获取成绩信息的时候有多次post，分别是获取某学期的成绩和获取总成绩；
  • 构造POST的思路：打开浏览器开发者工具，先正常走一遍查询成绩的流程，观察POST数据的key和value，并在代码中构造；
  • 由于页面返回的是一坨DIV，而且既没有name属性也没有id属性，所以需要将table标签整个转成array，并观察所需要的信息的位置，通过数组下标的方式访问；
  • 总的平均GPA无法区分主修专业和二专业/辅修专业，因为查询所有科目的总成绩页面中没有任何标签能区分这两者。

其他

  • 其实就是个爬虫，模拟了登陆正方教务系统并查询成绩的过程。Python写爬虫的话可能更方便，Python有个库requests，非常好用。
  • PHP中嵌入HTML感觉很不优雅，但HTML中嵌入PHP也好不到哪里去，有什么更好的办法吗？
  • 网上有很多关于正方教务的东西，虽然不太容易直接照搬，但学习价值很大。table转html的函数我就是从网上找的。
