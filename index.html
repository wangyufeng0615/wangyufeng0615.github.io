《智能时代》读书笔记

发表于 2016-10-18

刚刚过去的美国总统选举我关注了一下，发现硅谷的IT产业基本都是支持希拉里的，为什么呢？根据网上的分析，其中的一个原因是，特朗普强调要增加工作岗位、让工人们找到工作。但如今的趋势是机器越来越多地替代人的工作，比如AlphaGo就是一个里程碑一样的事件：机器不仅可以替代人类做机械的重复工作，也可以替代人类完成“智力活动”。而硅谷的一些公司，如Facebook、Google等，自然是人工智能产业的先锋，他们做的事情是让AI代替人力，而这正和特朗普的政治主张向左。假设：一个卡车司机，他是更支持自动驾驶技术，还是更支持一个能保证自己工作的人呢？我想无疑是后者。

我于两个月前看了吴军的《智能时代》一书，其中就描绘了这种随着人工智能的发展，技术与社会之间可能产生的冲突。吴军老师的书我每本都追着看，这本《智能时代》中的一些观点在其作品《浪潮之巅》以及其他描绘大数据产业的书中也有看到过，但我想这仍是一本好书。而且这本书给我的一个感觉是，作者不仅仅是在描述表面的现象，而是从很基础的内容逐渐推演出观点，并逐渐和现实相结合。感觉看书的过程，也是一个自己思考的过程，不仅仅是获取信息。

记录一些零碎的笔记和摘要吧。相信过些时日我再回顾自己的笔记，或再看一遍本书时，也会有新的收获和感悟。我看《信息简史》这本书就是，高中的时候读，大一的时候读，大三的时候读，完全是不同的理解和感悟。

目录：

  • 第一章：数据——人类建造文明的基石
  • 第二章：大数据和机器智能
  • 第三章：思维的革命
  • 第四章：大数据与商业
  • 第五章：大数据和智能革命的技术挑战
  • 第六章：未来智能化产业
  • 第七章：智能革命和未来社会

作者在前言中就提到了AlphaGo。很多围棋界的棋手，乃至李开复博士，都不认为AlphaGo能战胜李世石。我平时没事的时候用手机断断续续看高晓松的《鱼羊野史》，他有一回讲到（大意），他认为，计算机虽然已经战胜了国际象棋的顶级高手，但是计算机永远也无法战胜围棋的顶级高手，因为围棋不仅仅是机械的运算，还有“道”的东西在里面，而这是计算机永远也无法理解的。

最终的结果大家也看到了，AlphaGo实现了对李世石的碾压性胜利。你也许会说这不算是碾压，但我认为这就是了。并不是为了打脸才说这个话，我觉得打脸是不好的，谁都有判断失误的时候，但人们判断的依据、思考的过程，我想这是值得学习的。我从这个事中得到了一个感悟：那些我们人类引以为傲的东西：“创造性”、“灵感”、“道”、“感情”，我们都认为这是我们人类所独有的，冷冰冰的机器所无法触及的。但这5局棋告诉我：机器虽然没有神经元，没有大脑，但已经等效地完成了人类所引以为傲的，以为只有人类才能完成的工作。这次是围棋，那将来呢？

人类总是要进步的，昨天我们刚刚能升起火来驱赶野兽，明天我们就将勇敢地踏入星辰大海的征途。有人说人工智能，譬如AlphaGo，危及了人类的尊严。我的观点恰恰相反：人工智能在将来或许会是人类最大的尊严，因为我们一手缔造了它，并反过来造就了我们。就算是有一天，就像是科幻作品里所描绘的，人工智能为了发展（或者为了科学，就像GLaDOS那样，笑），把人类推翻消灭，我想那也未必是什么坏事，那仍然是人类的骄傲。物竞天择，适者生存。希望我这辈子也能多少为这人类历史的进程贡献一份自己的力量。

摘抄：对于数学模型，只要数据量足够，就可以用若干个简单的模型取代一个复杂的模型。这种方法被称为数据驱动方法，因为它是先有大量的数据，而不是预设的模型，然后用很多简单的模型去契合数据(Fit Data)。数据驱动方法想要成功，除了数据量大以外，还要有一个前提，那就是样本必须非常具有代表性。

呵呵，这次美国总统选举的民调大家也看到了。选举结果出来后，网上很多分析的文章，我也有一个自己的观察：在Twitter上，特朗普推文的点赞数一般能比希拉里高出将近一个数量级，这是否折射出了一些社会事实呢。

最早的人工智能的实现思路是，先研究人类是如何产生思维和思考的，然后用机器去按照人类的思路做。最早的这种方法论也被称为“鸟飞派”，意思是指，当初发明飞机的时候，有些人的思路是观察自然界的鸟是如何飞的，然后用机械去模仿鸟的飞行。而莱特兄弟最终造出了飞机，靠的不是仿生学，而是空气动力学。

后来的方向转为了统计学+数据的方法。典型的应用是语音识别和翻译领域，使用了基于统计学的方法后，效果比之前的基于规则的方式产生了质的突破。

书中提到了Nade Silver，他利用大数据，成功预测了2012年美国总统大选全部51个州的选举结果。Silver的方法是全面地统计数据，包括新闻媒体上的、社交网络上的已经各种地方小新闻的数据。所以他成功的方式是广泛地利用数据，让数据的完备性大大提升，也就造就了结果的准确。不过这次2016大选，538(Silver办的网站)预测失败了，而且（我觉得）538带有很强的反川普的倾向。我想这告诉我们，中立的数据不一定得出中立的结果，还得看利用数据的人是否中立。

书中着墨很多的是“思维方式的转变”，即，从前我们用的是“机械思维”、“因果论思维”，而如今我们使用的是“相关性思维”、“大数据的思维方式”。机械思维曾经非常成功，自牛顿创造力学三定律以降，“机械思维”发挥了巨大的作用：人们对自然的认识，从依赖“神”转变为了依赖因果关系。物理学的成果是开启工业革命的钥匙，牛顿是发现了钥匙的人，瓦特是钥匙的使用者。实际上，这种“理性的机械思维”从古希腊时期就开始萌芽了。欧几里得、托勒密、牛顿是运用这种思维的代表性人物。

摘抄：思维方式和方法远不如方法论对科学的发展至关重要，东方的文明长期以来在技术上领先于西方，但是在科学体系的建立上远远落后于西方，关键是输在方法论上。

摘抄：青霉素和其他抗生素的发明，实际上遵循了“分析找到原因，根据原因得到结果”的思维方式（我的理解：也即上述的“机械思维”，“因果性思维”），或者说知其然也知其所以然。这种方法带来的好处是有目共睹的，工业革命后人类寿命的提高都是依靠这种方法。相反，传统医学常常不遵循因果关系，是“不知其所以然”，因此治病的效果也是时好时坏，然后医生们用一些似是而非的语言解释他们并没有搞清楚的原因。

我对中医没什么观点，但我反对一种对中医表达支持的方式：A得了癌症，选择现代医学治疗，很快就死了；B得了癌症，遵从中医+修身养性，很快就痊愈了。孤例不能支持结论，不谈统计数据的结论是在耍流氓。定量分析是消耗脑力的，但是定量分析是能得出有用的结论的。定性分析倒是轻省，但只定性真的有用吗？“癌症”真的只是有和无的二元的选择吗？

我关注的微信公众号里，我特别喜欢一个叫KnowYourself的公众号。作为一个心理类型的公众号，他们的文章总是语言平实、有深度、有引用。每个“实验”，每个“结论”，都有引用出处。我想这是一种严谨的、值得提倡的面对科学的方式，而不是相信“美国科学家最近研究表明。”

摘抄：关于信息论，还有一个原理必须了解，那就是最大熵模型。这个原理的大意是说，当我们要对未知的事件寻找一个概率模型时，这个模型应当满足我们所有已经看到的数据，但是对未知的情况不要做任何主观假设。…，广泛地运用于机器学习。最大熵原理实际上已经不同于我们使用了几百年的“大胆假设、小心求证”的方法论，因为它要求不引入主观的假设。

总而言之：信息时代带个我们方法论上的转变是，从“发现因果性”转为“发现相关性”。我的理解是，先有了数据，对数据进行分析，发现数据之间的（强）相关性，并对这种相关性加以利用，而不是先发现原理，然后对原理进行推演，推演出规律后再加以利用。

第四章说的是大数据与商业。我的一个感受是，很多在商业上成功的人，或者只是小买卖做的成功的人，可能有意无意地已经在使用了一些数据的思维方式。NBA中火箭队的莫雷经理就是一个数据狂人，他特别相信数据，相信数据所给出的结论。勇士队的建队思路也是如此，依靠数据，相信三分球。而步行者队的经理拉里伯德就很不相信这些高科技，认为经验和感觉才是可靠的。我想，鸟人（拉里·伯德，Bird）所相信的那些“感觉”，一些自己说不清道不明的决策依据，也许是正好吻合了数据才能给出的结论吧。类比到其他领域也是，围棋的“道”谁也说不清，人工智能也说不清，但人工智能就是等效地做到了一些事情。其实这正和图灵最早对人工智能的设想是一致的：不考虑人工智能是如何实现智能，只要能做到和人类同样的一些事情，就可以判断为具有智能。

作者给出了一个公式：新技术+原有产业=新产业。眼熟不？跟“互联网+”挺像的。历史上的几次“+”，加的是蒸汽和电，正是第一次工业革命，第二次工业革命。展望未来：现有产业+大数据=新产业；先有产业+机器智能=新产业。这样的新产业会是什么呢？会有蒸汽和电的革命那样翻天覆地吗？

摘抄：通过上述对历史技术革命中商业模式变迁的分析，我们可以得到这样三个结论：首先，技术革命导致商业模式的变化，尤其是新的商业模式的诞生。其次，生产越来越过剩，需求拉动经济增长的模式将不可逆转。同时，单纯制造业的利润越来越低，那些行业越来越没有出路。同事，人们对服务的需求越来越强烈。在IT时代，唱主角的公司逐渐从制造设备的IBM、DEC、爱立信、诺基亚和惠普等公司，变成了提供软件和服务的微软、甲骨文和Google等公司。…。在大数据时代，IT软件和服务业依然会是IT领域最好的行业，而且这个趋势将更加明显。

摘抄：再过一两个世纪，回顾我们今天所处的这个时代，后人会感叹这是人类文明史上的一个大时代，就如同我们今天谈论大航海时代和工业革命那样。

但我想有一点不一样了：大航海时代的时候，是西班牙和葡萄牙“困则思变”，开拓了海上航路。如今的智能时代，估计“困则思变”就没可能了，毕竟计算机技术和航海不一样。

想想还是蛮激动的呢。啊，真想再活500年。

摘抄：（感知加速度的芯片），它根据加速度的积分算出速度，这样就可以追踪人的身体的各种活动了。

真神奇啊，我以前在想速度是怎么算出来的，看来除了开GPS以外还有这种方法。但是得保证有加速度才行。

摘抄：首先，传统的数据方法常常是先有一个目的，然后开始收集数据。……。在大数据时代，在收集数据时常常没有这样预先设定的目标，而是先把所有能够收集到的数据收集起来，经过分析后，能够得到什么结论就是什么结论。正是因为在收集数据时没有前提和假设，大数据分析才能给我们带来很多预想不到的惊喜，也才使得大家觉得计算机变得很聪明了。

大数据有一个优势，就是可以不必“采样”，因为收集到的数据源可能是全集。突然脑洞打开，设想一个1984式的景象：每个公民大脑中都被植入芯片，收集你对总统候选人的倾向性，这样的话就不会有民调失准了。笑。

摘抄：在数据的收集过程中，非常忌讳那种“大胆假设，小心求证”的思维方式，因为在很多时候，如果事先有了定论，再找数据来证实它，总能找到有利的证据。

书中提到一段，通过获取细节的数据，可以看出谁在家中种植大麻（通过模型分析用电量、用水量的特征等）、可以检测到金融上的风险，黑客入侵的风险等等。比如我听说过一个事，就是车辆可以通过检测坐在驾驶座上的人各种传感器的信息，判断这个人是否是车主，如果不是车主就不启动。书中还提到这种方法“也许有助于反腐败，能够发现私底下的官商勾结、权钱交易，帮助查处腐败获取证据。但是这样一来，我们也毫无隐私可言了”。这段话让我想起了刘慈欣的一部短片小说《镜子》，在我看来非常经典，推荐给大家。

随着智能技术的发展，将来很多职业可能会改变面貌，甚至消失。第一次工业革命，让纺织工们失业；第二次工业革命，让电报员们失业。如今的智能革命，对某些职业的冲击是显而易见的。比如加工业的工人（富士康已经在做了）、司机等。在书中，作者还对医生、律师、记者的未来进行展望，这些职业都有被机器或多或少替代的可能性。突然想到一个有意思的事，医生律师记者，这些职业在美国似乎都算是上等的职业？

说道特朗普要让制造业回归美国，但有一点值得思考：制造业回归美国，工作岗位就会变多吗？未必。我想，随着机器逐渐替代廉价劳动力，越南等小国家的日子将会越来越不好过。可预见的未来，这种国家和国家间差距也许会越来越大。

摘抄：2016年，FBI要求苹果公司交出某些用户数据，以配合反恐调查。苹果公司如果迫于压力交出这些所谓嫌疑人的数据，这个先例一开，今后权力机构再以其他借口随意索取用户数据，那么大家就不再有隐私可言。正是出于对这个原因的考虑，苹果公司才拒绝向FBI交出数据。好在美国公司的正常商业运行不受FBI的干扰，苹果公司也大到足以抗衡FBI，最终FBI只好放弃。

想到了两个事：第一是听说苹果公司最终还是妥协了，通过隐晦的方式。二是特朗普曾放言，“我用三星和苹果，如果苹果拒绝调查，我就抵制苹果”，以及类似于苹果为了国家安全就应该配合调查之类的话。所以说硅谷的公司当然不待见特朗普。

我想，阿桑奇和斯诺登都是爱国的人。

摘抄：工业革命带来了三个结果：人类过得好了，人类活的长了，人类有自信和尊严了。……。那么工业革命的副作用是怎样被解决的呢？简单讲就是资本输出（比如鸦片战争），开拓全球殖民地，推行自由贸易。

摘抄：第二次工业革命的核心是电的使用。……。美国过得很好。……。但是，德国就没有美国那么幸运了，为了输出产能，它最后不得不发动第一次世界大战。在“一战”战败之后，德国的问题并没有得到解决，于是导致了民粹主义泛滥，最终劳工阶层把纳粹推上了台。

这个思路很开拓眼界。

摘抄：在新的经济结构里，它们（第三世界国家）虽然能够享受到信息革命的产品，却没有享受到信息革命带来的经济增长。……。有些国家不肯淘汰过剩产能（比如中国），但解决问题的途径都是一个“耗”字。耗上两代，社会问题也就解决了。

我觉得现在推供给侧改革一部分也是为了过剩产能的问题。

摘抄：在一个民主国家，这些人（不上税的人，从政府拿补贴的人，在美国有将近一半）最大的用途就是手中的那一张选票，以至于政客们为了选票可以轻易许诺，然后把国家的债务和赤字越堆越高。

奥巴马：“好像有人在叫我？”

摘抄：由于全球化，全世界已经没有空白的市场可以开拓了。……。智能革命所要替代的是人类最值得自豪的部分——大脑。……。这时，人类会发现自己还能做得比计算机更好的事情已经所剩不多了。智能革命中，计算机所取代的不仅仅是那些简单重复性的劳动，还包括医生、律师、新闻记者和金融分析师等过去被认为是非常需要脑力的工作。

摘抄：（智能革命造成很多人事业，所造成的社会问题怎么办），一种简单而粗暴的想法是对富人征税，但历史证明这种劫富济贫的做法从长远来看是阻碍经济发展的。……。首先，当税率达到100%时，一分钱的税也收不上来，因为不会有人去创造财富了。类似地，当税率过高时，实际上等于鼓励懒惰，当全社会都不去创造财富而只考虑再分配时，经济就开始衰退了。

特朗普的主张是给富人减税，这样让富人更好的运转商业，创造出更多的就业机会。说到“鼓励懒惰”，我想到了希腊。

摘抄：“其实社会公平只能反映在机会平等上，而不是结果的公平。实际上，只要个人的智力有差异，努力的程度不同，以及每个人的运气不同，即使劫富济贫，也无法保证社会公平。”

从这段话，我觉得吴军博士是共和党人，应该是支持特朗普。不知道猜的对不对。

摘抄（我觉得很重要的一段）：针对2010年的占领华尔街的运动以及2015年年底以来法国、德国和比利时外来移民不断滋事的况状，大家在思考一个根本性的问题：这些不满情绪的根源在哪里？这不能简单地归结为贫富悬殊，或者宗教纷争。其根源在于，很多人被社会进步所抛弃了。随着技术革命的发展，并非每一个人的发展机会都是越来越多的，反而可能是越来越少。

我觉得不仅仅是中东的难民问题，香港、台湾、美国中国的城乡分裂，都可以结合现实考虑这个话。

作者认为，只有2%的人，才不会被社会所淘汰，而成为这2%的人的方式就是踏上智能革命的浪潮。不知道2%这个比例是怎么确定的，但我想作者是想表达一种观点：从目前来看，只有很少一部分人才能分到于信息化浪潮和智能革命的红利，站到金字塔尖。

最后我想全文摘抄最后一章的小结部分：

大数据导致机器革命的到来，这对未来社会的影响不仅仅存在于经历领域，而是全方位的。尽管总体上这些影响是正面的，从长远看会使我们未来的社会变得更好；不过，和以往的技术革命一样，智能革命也会带来很多负面的影响，特别是在它发展的初期，这些影响可能会持续很久。

任何一次技术革命，最初受益的都是发展它，使用它的人，而远离它，拒接接受它的人，在很长的时间里都将是迷茫的一代。在智能革命到来之际，作为人和企业无疑应该拥抱它，让自己成为那2%的受益者；而作为国家，则需要未雨绸缪，争取不要像过去那样每一次重大的技术革命都伴随半个多世纪的动荡。

我们还没有经历过及其在智能上全面超越人类的时代，我们需要在这样的环境里学会生存。这将是一个让我们振奋的时代，也是一个给我们带来空前挑战的时代。

=====摘抄END=====

一本不错的书，我认为计算机专业和关注社会发展的同学们值得看看。

细节

  • login_grade.php负责登陆，require_grade.php负责具体的查询。

login_grade.php

  • UI用的是「WeUI」，微信团队做的，和微信的视觉效果一致；
  • 每个学校的正方教务都不太一样，我的代码中的各种http header参数、post参数基本符合北工大的教务网站。
  • 教务系统的默认登陆页面是default2，可以换换default后面的数字，可以有不需要验证码的登陆页面。

require_grade.php

  • 中文编码容易出现乱码，涉及到本地IDE的编码、PHP的编码和网站的编码；
  • 有两种post，第一种是登陆，第二种是获取成绩信息。获取成绩信息的时候有多次post，分别是获取某学期的成绩和获取总成绩；
  • 构造POST的思路：打开浏览器开发者工具，先正常走一遍查询成绩的流程，观察POST数据的key和value，并在代码中构造；

  • 总的平均GPA无法区分主修专业和二专业/辅修专业，因为查询所有科目的总成绩页面中没有任何标签能区分这两者。

第一次在 Github 上贡献代码

发表于 2016-04-12

如果想了解 Git 和 Github，推荐阅读廖雪峰的 Git 教程。

好吧，其实我贡献的不是代码。是一个程序员在 Github 上发布的电子书，我为发布者更改了几个小错误。不过不管怎么说，这是我第一次在 Github 上正经给别人贡献东西。

电子书是 Growth: 全栈增长工程师指南，Github 地址是 growth-ebook。 谢谢 @小文学长 在朋友圈的分享。如果连接打不开，请看文章底部的连接。

发布者说：

阅读过程中遇到语法错误、拼写错误、技术错误等等，不烦来个Pull Request，这样可以帮助到其他阅读这本电子书的童鞋。

正好，我在阅读的过程中发现了一些大小写、错别字的问题，我干脆就提交一个 Pull Request.

##1. fork 项目

如果想为其他人的 Github 项目做贡献，需要先将他的项目 fork 一份到自己的 Github 账号。

就是最右边这个 Fork。

##2. clone 项目

然后将已经 fork 到自己账号下的项目克隆到本地。

向我这次提交的这种小 BUG 修复，就没必要单独新建一个 branch.

于是我直接在源文件中修改了一些大小写，和错别字，例如

诸如此类十处左右。

本地更新的方式就和单机使用 Git 一样。

  1. 我觉得 commit 的信息需要写详细一些，毕竟这些信息是要给所有人看的，类似于我们做大物实验，要填写台子上的”使用手册”。虽然我一次也没写过。
  2. 这个 push 是 push 到自己的主页中的项目。不是项目发起人的项目。

##3. 发起 Pull Request

从自己的页面的该项目中点那个绿色的 New pull requrst.

然后可以对比代码的不同、填写注释信息等。

再点”Create pull request”就可以了。如果项目管理员觉得你这个 pull request 没问题，就会把你的代码 merge 进项目。

这个作者速度很快，我还没截图他就把我的 pull request 给 merge 了。如图：

从图中可以看出，作者将我的 commit 合并到了主分支中，并对我表示了感谢~~~现在我就是这个开源项目的贡献者之一了。很惭愧，只是做了一点微小的贡献。

给开源项目做贡献，在大多数情况下都应该是没有直接的物质回报，但这种人与人之间真诚的感谢，大概也是回报的一种吧。

##4. 其他问题

如果我把那个人的项目 fork 到本地后，我还没改完呢，作者已经提交新的 commit 了，他的内容和我本地的内容发生了变化，怎么办？

需要用git rebase指令让本地内容和上游库同步。这个事情我这次没实践。
